# PR Description Template

## Summary

- What changed at a high level.

## Motivation

- Why this change is needed.

## Changes

- Main code, test, docs, config, or migration changes.

## Validation

- Commands run and results.

## Risks

- Known risks, weak coverage, or behavior that needs attention.

## Rollback

- How to revert or disable the change if needed.

## Screenshots Or Examples

- Include screenshots, curl output, logs, or examples when relevant.

## Checklist

- [ ] Diff is focused.
- [ ] Tests or relevant validation were run.
- [ ] Docs or examples were updated when needed.
- [ ] Migration or rollback notes are included when needed.
- [ ] No secrets or local machine paths are included.
