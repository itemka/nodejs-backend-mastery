# Debug Template

## Goal

Describe the failure or unexpected behavior.

## Context

Explain when it started, what changed recently, and who is affected.

## Exact Error

Paste the command, stack trace, log excerpt, or observed symptom.

## Reproduction

List exact steps, input data, environment, and expected vs actual result.

## Files Involved

List suspected files, tests, packages, services, or configs.

## Expected Output

Ask for root cause, minimal fix, regression test, or investigation notes.

## Validation Required

List the command or scenario that must pass after the fix.

## Risks

List fragile areas, external services, data risks, or constraints.
