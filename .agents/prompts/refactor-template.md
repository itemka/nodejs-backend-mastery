# Refactor Template

## Goal

Describe the internal structure you want improved.

## Context

Explain why the current structure is hard to maintain or extend.

## Behavior To Preserve

List public APIs, response shapes, tests, fixtures, and user-visible behavior that must not change.

## Constraints

List boundaries, dependency limits, timeline, or compatibility constraints.

## Files Involved

List target modules and nearby tests.

## Expected Output

Ask for a small refactor plan or implementation with before/after notes.

## Validation Required

List characterization tests, existing tests, typecheck, lint, or manual checks.

## Risks

List regression-prone paths, shared modules, or areas with weak tests.
