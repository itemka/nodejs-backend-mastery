# Code Review Template

## Goal

Request a review of a diff, branch, PR, or set of files.

## Context

Explain the purpose of the change and any relevant issue or acceptance criteria.

## Scope / Non-Goals

List files, behavior, or review areas that should stay out of scope.

## Review Focus

List focus areas such as correctness, security, API contract, data migration, typing, tests, or frontend UX.

## Files Involved

List important files or modules, or say to inspect the current diff.

## Expected Output

Ask for findings ordered by severity with concrete file-level references.

## Validation Required

List checks already run and checks the reviewer should consider.

## Risks

List known weak spots, shortcuts, or areas that need extra scrutiny.

## Final Response Needs

Ask for findings first, ordered by severity, with file and line references where possible.
