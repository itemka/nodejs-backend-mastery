# Task Template

## Goal

Describe the behavior or outcome you want.

## Context

Explain why this matters and what currently happens.

## Constraints

List compatibility, architecture, dependency, performance, security, or timeline constraints.

## Files Involved

List known files, modules, routes, packages, or docs.

## Expected Output

Describe the desired code, docs, tests, or analysis result.

## Validation Required

List commands, tests, manual checks, or acceptance criteria.

## Risks

List risky migrations, public API changes, data loss risks, or unknowns.
