# Architecture Decision Template

## Goal

Describe the architecture decision needed.

## Context

Explain current architecture, business need, and why a decision is needed now.

## Non-Goals

List what should not change as part of this decision.

## Options Considered

List known options and any rejected ideas.

## Constraints

List scalability, consistency, security, cost, operational, team, and compatibility constraints.

## Files Involved

List existing modules, docs, schemas, configs, or prototypes.

## Expected Output

Ask for a recommendation with trade-offs and implementation implications.

## Validation Required

List tests, benchmarks, rollout checks, or proof points needed.

## Risks

List migration risks, lock-in, failure modes, unknowns, and rollback concerns.

## Final Response Needs

Ask for recommendation, rationale, trade-offs, affected files, and next validation step.
